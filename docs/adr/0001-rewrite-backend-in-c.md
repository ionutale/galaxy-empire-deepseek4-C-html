# Rewrite backend from Go to C

The backend was originally written in Go 1.26 using chi, pgx, and golang-jwt. We decided to rewrite it entirely in C (libuv + llhttp + libpq) to minimize memory footprint and maximize request throughput. The Go runtime's goroutine overhead and GC latency were not justified for this game's workload — most requests are simple DB lookups and JSON serialization, where C's predictable allocation model and sub-millisecond startup time provide measurable wins. The trade-off is significantly more manual memory management and a longer initial build, but the request-per-second ceiling and container memory budget (~8MB RSS vs ~30MB for the Go binary under load) make it worthwhile. We accepted that some features (e.g., `pgx` connection pool, `golang-jwt`, chi's route groups) will need to be rebuilt from scratch in C, and that the first vertical slice (auth + planet dashboard + resource tick) will take longer than a Go equivalent would.

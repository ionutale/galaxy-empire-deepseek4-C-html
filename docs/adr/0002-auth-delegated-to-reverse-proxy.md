# Delegated authentication to Nginx/OpenResty reverse proxy

JWT validation was handled by the Go backend (golang-jwt middleware). We decided to offload it entirely to Nginx/OpenResty with lua-resty-jwt so the C backend never parses tokens. Nginx validates the JWT signature and expiry, extracts the user_id claim, and forwards it as an `X-User-ID` header to the upstream C process. This keeps the C codebase purely focused on game logic (no crypto libraries, no token parsing, no key management), improves request throughput by validating at the edge, and lets us swap auth providers without touching application code. The trade-off is that OpenResty's Lua scripting adds a config-layer dependency, and the C backend must trust the `X-User-ID` header implicitly — which is safe only because Nginx runs in the same Docker network and strips any incoming `X-User-ID` from external clients before validation.
